# Реверс-инжиниринг Patisson Game — как исходники восстановлены из `.exe`

Эта ветка — **эксперимент**: восстановление исходного кода игры из дистрибутива
`Patisson_Setup.exe` без доступа к оригиналу. Здесь описан весь путь, инструменты
и, главное, как результат был **проверен на точность**.

> ⚠️ Это учебный/исследовательский разбор собственной игры (с разрешения автора).
> Оригинальный авторский код — в ветке `main`.

## Цепочка упаковки

```
Patisson_Setup.exe
  │  Inno Setup 6.5.0 (инсталлятор)
  ▼
Patisson.exe                     ← PyInstaller one-file, Python 3.12, +Ursina/Panda3D/NumPy/Pillow
  │  CArchive + PYZ
  ▼
patisson.pyc                     ← скомпилированный байткод игры (главный модуль)
  │  декомпиляция Python 3.12
  ▼
patisson.py                      ← восстановленный исходный код
```

## Шаги

### 1. Распаковка инсталлятора (Inno Setup 6.5)
`innoextract` не поддерживает Inno Setup 6.4+ («Unexpected setup loader revision: 2»),
а 7-Zip 23.01 не открывает этот формат. Установщик распакован запуском в тихом
режиме (`/VERYSILENT /SUPPRESSMSGBOXES /DIR=…`) в отдельную папку. Внутри —
`Patisson.exe` (255 МБ) и папка `resources/` с ассетами.

### 2. Распаковка PyInstaller
Сигнатуры в `Patisson.exe`: `PKG archive!`, `python312.dll`, `numpy._core`. Это
**PyInstaller one-file, Python 3.12**. Вскрыто `pyinstxtractor-ng` → точка входа
`patisson.pyc` (101 КБ) + библиотеки (ursina 7.0.0, panda3d 1.10.15, numpy 2.3.1,
PIL) — библиотеки открытые, их не трогаем.

### 3. Декомпиляция байткода Python 3.12
Готовых надёжных декомпиляторов под 3.12 нет: `uncompyle6`/`decompyle3`
останавливаются на 3.9. Использован **`pycdc` (Decompyle++)**, собранный из
исходников (CMake + MSVC) и **пропатченный** под конструкции 3.12, на которых он
падал или которые игнорировал:

- `CALL_INTRINSIC_1` (`INTRINSIC_IMPORT_STAR`) — `from ursina import *`;
- `POP_JUMP_IF_NONE` / `POP_JUMP_IF_NOT_NONE` — проверки `is None`;
- `MAKE_CELL`, `COPY_FREE_VARS`, `RETURN_GENERATOR` — замыкания и генераторы;
- `LOAD_FAST_CHECK` — возможно-несвязанные локальные.

Инлайн list-comprehension (`LOAD_FAST_AND_CLEAR`) в 3.12 — нерешённая для pycdc
проблема; единственный такой блок в модуле (расстановка забора) временно вырезан
из байткода **хирургией `.pyc`** (NOP-замена + перестройка таблицы исключений),
чтобы pycdc прошёл модуль целиком. Сам comprehension вписан обратно вручную.

### 4. Ручная реконструкция и исправление багов pycdc
pycdc на 3.12 местами выдаёт **синтаксически валидный, но логически неверный**
код. Найденные и исправленные классы багов:

- **инверсия булевых условий** — цепочки `A or B or C` он рендерит как
  `A and B and C` (ломало `update()`, `input()`, `toggle_menu`, `toggle_dev_panel`,
  `start_fishing`, `start_harvest`, `open_shop`, `apply_cheat`, `validate_cheat`);
- **`obj.attr += …`** (присваивание-с-операцией к атрибуту) — выдавал мусор;
- **генераторные выражения** (`next(… for …)`, `'\n'.join(… for …)`);
- **`with open(...)`** — путал структуру (`save_settings`/`load_settings`);
- **аргумент по умолчанию** `state=None` превращался в `state=(None,)`;
- **`lambda: None`** рендерился как невалидный `lambda: pass`.

Все такие места восстановлены по дизассемблировке (`patisson_disassembly.txt`) и
помечены в коде комментарием `# [reconstructed]`.

## Проверка точности (главное)

Чтобы не отдать «правдоподобный, но неверный» код, написан **верификатор**
(`tools/verify.py`): он берёт восстановленный `.py`, перекомпилирует каждую
функцию Python 3.12 и сверяет её байткод с оригиналом **поинструкционно, включая
направление условных переходов** (именно это ловит инверсии булевых условий).

**Результат: все 85 функций совпадают с оригиналом байт-в-байт.** Тело модуля
(`tools/module_verify.py`) тоже совпадает — кроме доказуемо эквивалентной
косметики: цепочки `a = b = X` (декомпилятор разбивает на строки), `1` vs `1.0`,
и `fence_positions` записан как `c1 + c2 + …` вместо `[*c1, *c2, …]`.

## Файлы в этой папке

| Файл | Что это |
|------|---------|
| `patisson.py` | Восстановленный исходник (результат) |
| `patisson_disassembly.txt` | Полная дизассемблировка байткода (эталон) |
| `assets_manifest.txt` | Перечень ресурсов с размерами |
| `tools/verify.py` | Верификатор: перекомпиляция + сверка байткода функций |
| `tools/module_verify.py` | Сверка кода уровня модуля |
| `tools/assemble.py` | Сборка финального `.py` (вывод pycdc + ручные правки) |
| `tools/surgery.py` | Хирургия `.pyc`: вырезание инлайн-comprehension |
| `tools/pycdc_patch.md` | Что именно добавлено/исправлено в pycdc |

## Воспроизведение

1. Распаковать установщик (тихая установка) → `Patisson.exe`.
2. `pip install pyinstxtractor-ng` → распаковать → `patisson.pyc`.
3. Собрать pycdc с патчами (см. `tools/pycdc_patch.md`).
4. `python tools/surgery.py` → `patisson_surgery.pyc`; `pycdc` → черновой `.py`.
5. `python tools/assemble.py` → финальный `patisson.py`.
6. `python tools/verify.py` → должно быть `85 / 85`.
